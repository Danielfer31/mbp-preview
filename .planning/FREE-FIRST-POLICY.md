# Politica De Herramientas Gratuito-Primero

## Objetivo

Evitar costos innecesarios sin aceptar riesgos desproporcionados de privacidad, seguridad,
continuidad o mantenimiento.

## Orden De Preferencia

1. Capacidades existentes que no requieran herramienta nueva.
2. Software libre mantenido y desplegable de forma sostenible.
3. Plan gratuito con limites claros, exportacion y salida viable.
4. Servicio pago con beneficio, presupuesto y aprobacion demostrables.

## Costo Total De Propiedad

Toda evaluacion registra:

| Dimension | Pregunta obligatoria |
|---|---|
| Licencia | Cuanto cuesta usar legalmente la herramienta |
| Infraestructura | Donde corre y cuanto cuesta alojarla |
| Operacion | Cuantas horas mensuales requiere mantenerla |
| Seguridad | Quien actualiza, monitorea y responde incidentes |
| Datos | Que datos recibe, donde residen y como se eliminan |
| Dependencia | Se pueden exportar datos y reemplazar el servicio |
| Escalamiento | Que limite dispara cobro o migracion |

## Puerta De Adopcion

Una herramienta nueva se aprueba solo si:

- Resuelve un requisito activo, no uno hipotetico.
- Tiene propietario y responsable de salida.
- Se verificaron terminos, limites y precios en fuente oficial.
- Se documento una alternativa y el costo de migracion.
- No crea tratamiento de datos innecesario.
- El costo total cabe en el escenario presupuestal aprobado.

## Reglas

- No ingresar tarjeta para activar una prueba sin aprobacion humana.
- No asumir que un plan gratuito permanecera igual; revisar trimestralmente.
- Configurar alertas o controles para impedir cobros sorpresa cuando sea posible.
- No usar herramientas gratuitas que moneticen datos de pacientes o visitantes.
- Preferir despliegues estaticos porque reducen costo, mantenimiento y superficie de ataque.
- Una herramienta self-hosted no se clasifica como costo cero si requiere servidor o soporte.

## Revision

El registro de herramientas se revisa antes de cada fase y cada tres meses durante operacion.
