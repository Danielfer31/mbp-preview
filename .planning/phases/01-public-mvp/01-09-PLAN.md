<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI -> Cloudflare Pages | el token de despliegue puede modificar el sitio en vivo |
| build -> produccion | un build inseguro no debe auto-desplegar |
| sitio en vivo -> visitante | headers/CSP reales deben coincidir con lo definido |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-24 | Tampering | auto-deploy desde build inseguro (H3) | mitigate | deploy solo tras build:production; rama de produccion protegida |
| T-01-25 | Elevation | token de Cloudflare con exceso de permisos (H3) | mitigate | token minimo privilegio (Pages Edit), MFA, secrets fuera de Git |
| T-01-26 | Tampering | headers no aplicados en vivo (H4) | mitigate | post-deploy-verify contra la URL real |
| T-01-27 | Availability | deploy roto sin reversion (H4) | mitigate | procedimiento de rollback con rehearsal y rebuild reproducible |
| T-01-28 | Tampering | dependencia vulnerable en produccion | mitigate | npm audit del lockfile de produccion antes de desplegar |
</threat_model>
